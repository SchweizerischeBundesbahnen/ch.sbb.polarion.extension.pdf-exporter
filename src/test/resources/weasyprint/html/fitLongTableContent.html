Table Overflow with /-123 Pattern Cause

WeasyPrint delegates line breaking to Pango, which follows the Unicode Line Breaking Algorithm (UAX #14). Four rules combine to eliminate all break opportunities at spaces when followed by /, -, and digits:

Rule Char Class Effect
LB7: × SP (space) SP No break before a space
LB13: × SY / SY (Symbol) No break before /
LB21: × HY - HY (Hyphen) No break before -
LB25: HY × NU - + digit HY, NU No break between - and digits

How the space gets trapped

At the boundary between two /-123 groups:

...3 /-1...

  3 → (space) : LB7 blocks  -- no break before space
  (space) → /  : LB13 blocks -- no break before /
  / → -        : LB21 blocks -- no break before -
  - → 1        : LB25 blocks -- no break between hyphen and digit

Every position around the space is blocked by a different rule. The entire text "/-123 /-123 /-123..." becomes one unbreakable unit.

Note: CSS Text Level 3 (section 6.2) mandates that collapsible spaces always provide soft wrap opportunities. Pango follows strict UAX #14 without this CSS override.

Consequence

The min-content width of each table cell becomes the width of the entire text (all 14 tokens) instead of just one /-123. Three such cells exceed the page width, and the table layout algorithm forces the table wider than the page.
